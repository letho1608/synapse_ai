22 Model HF
Layer Sharding Phân tán theo layer
OpenAI API tương thích

Mục tiêu dự án

1
Triển khai LLM phân tán — Chạy mô hình ngôn ngữ lớn trên nhiều máy, chia tải theo layer (sharding) để tận dụng RAM/GPU của từng node.
2
Cung cấp API chuẩn OpenAI — Endpoint /v1/chat/completions (streaming/non-streaming) để ứng dụng bên ngoài tích hợp chat mà không phụ thuộc nhà cung cấp.
3
Quản lý model & data tập trung — Tải/xóa 22 model từ Hugging Face từ giao diện web; quản lý dataset (JSON/JSONL), fine-tune LoRA và Chat với Model ngay trên dashboard.
4
Cluster linh hoạt — Kết nối nhiều node qua Tailscale, UDP hoặc cấu hình thủ công; discovery, topology và partitioning tự động theo tài nguyên (CPU, RAM, FLOPS).

Chức năng chính

Inference phân tán

Chạy model PyTorch/HF trên CPU/GPU; chia layer qua nhiều node (sharding), gRPC + retry, đồng bộ topology.

API Chat (OpenAI)

/v1/chat/completions streaming hoặc non-streaming; tích hợp client bên ngoài như OpenAI SDK.

Quản lý mô hình

22 model Hugging Face: danh sách đã tải / có thể tải; tải (snapshot_download) và xóa (HF cache) từ web UI.

Cluster & Discovery

Tailscale, UDP, Manual; thu thập topology (CPU, RAM, FLOPS); partitioning (RingMemoryWeighted) gán layer theo node.

Dataset

Tạo/upload/tải/xóa/đổi tên file JSON/JSONL; kiểm tra format, xem trước; dùng cho training LoRA.

Training LoRA

Chọn model, dataset, epochs, batch size; chạy fine-tune LoRA từ Dashboard; theo dõi trạng thái và tiến độ.

Chat với Model

Trò chuyện với model đã tải, giao diện chat kiểu ChatGPT; chỉnh temperature, xem usage token; Enter gửi, model mặc định từ Cài đặt.

Giám sát

Danh sách node Tailscale, trạng thái phân tán (partitions), thống kê hệ thống (CPU, RAM, uptime).

Kiến trúc phân tán

Một request chat đi qua pipeline layer sharding: node layer đầu encode prompt và chạy các layer thuộc shard của mình → output tensor gửi (gRPC) sang node tiếp theo → … → node layer cuối sample token và stream về API. Partitioning (vd. RingMemoryWeighted) chia tỷ lệ layer theo RAM/FLOPS. gRPC có retry (tối đa 3 lần); đồng bộ topology định kỳ và broadcast trạng thái (opaque status, result).
